## Управление пользователями

В этой задаче вам нужно создать bash-скрипт для управления пользовательскими аккаунтами в системе. Скрипт должен предоставлять различные опции для работы с пользователями через аргументы командной строки.

### Часть 1: Создание аккаунта

1. Реализуйте опцию `-c` или `--create` для создания нового пользователя. Скрипт должен запрашивать имя пользователя и пароль.
2. Проверяйте, существует ли уже такое имя пользователя. Если да — покажите сообщение об ошибке и завершите работу.
3. После успешного создания покажите сообщение с именем нового пользователя.

### Часть 2: Удаление аккаунта

1. Реализуйте опцию `-d` или `--delete` для удаления существующего пользователя. Скрипт должен запрашивать имя пользователя для удаления.
2. Проверяйте, существует ли пользователь перед удалением. Если нет — покажите сообщение об ошибке.
3. После удаления покажите подтверждение с именем удаленного пользователя.

### Часть 3: Сброс пароля

1. Реализуйте опцию `-r` или `--reset` для сброса пароля существующего пользователя. Скрипт должен запрашивать имя пользователя и новый пароль.
2. Проверяйте существование пользователя перед сбросом пароля.
3. После сброса покажите сообщение с именем пользователя и новым паролем.

### Часть 4: Список пользователей

1. Реализуйте опцию `-l` или `--list` для вывода всех пользователей системы. Показывайте имена пользователей и их UID.

### Часть 5: Справка

1. Реализуйте опцию `-h` или `--help` для показа информации об использовании скрипта и доступных опциях.

### Бонусные задачи (опционально)

Для дополнительной практики можно добавить:
  - Подробную информацию о пользователях (домашнюю директорию, оболочку и т.д.)
  - Возможность изменения свойств аккаунта (имени, UID и т.д.)
Важно: Обрабатывайте ошибки корректно, добавляйте понятные подсказки и комментируйте код.

## Инструкция по отправке

Создайте bash-скрипт user_management.sh с реализацией всех перечисленных функций. 
Добавьте комментарии, объясняющие логику каждой части.

## Задача недели 3: Автоматическое резервное копирование с помощью Cron

Это вторая задача для Дня 2 Bash Scripting Challenge! Вам нужно создать bash-скрипт для резервного копирования указанной директории с механизмом ротации бэкапов.

## Описание задачи

Создайте bash-скрипт, который принимает путь к директории как аргумент командной строки и создает ее резервную копию. Скрипт должен создавать папки для бэкапов с временными метками и копировать все файлы из указанной директории.

Дополнительно, скрипт должен реализовать механизм ротации, сохраняя только последние 3 бэкапа. Если папок с бэкапами больше 3, самые старые должны удаляться.

> Скрипт создает папку для бэкапа с временной меткой внутри указанной директории и копирует туда все файлы. Также он проверяет существующие бэкапы и удаляет самые старые, оставляя только 3 последних.

## Пример использования

Предположим, скрипт называется backup_with_rotation.sh. Пример выполнения в разные даты:

1. Первый запуск (2023-07-30):

```
$ ./backup_with_rotation.sh /home/user/documents
```

Вывод:

```
Backup created: /home/user/documents/backup_2023-07-30_12-30-45
Backup created: /home/user/documents/backup_2023-07-30_15-20-10
Backup created: /home/user/documents/backup_2023-07-30_18-40-55
```

После этого в /home/user/documents будут:

```
backup_2023-07-30_12-30-45
backup_2023-07-30_15-20-10
backup_2023-07-30_18-40-55
file1.txt
file2.txt
...
```

2. Второй запуск (2023-08-01):

```
$ ./backup_with_rotation.sh /home/user/documents
```

Вывод:

```
Backup created: /home/user/documents/backup_2023-08-01_09-15-30
```

Теперь в /home/user/documents будут:

```
backup_2023-07-30_15-20-10
backup_2023-07-30_18-40-55
backup_2023-08-01_09-15-30
file1.txt
file2.txt
...
```

Скрипт создает бэкапы с именами, содержащими временные метки, и сохраняет только 3 последних бэкапа, удаляя более старые.

## Инструкция по отправке
